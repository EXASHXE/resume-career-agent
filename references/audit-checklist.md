# Resume Audit Checklist

对每项标记 `通过 / 部分通过 / 缺失 / 待确认` 并引用原文证据。

- 职业故事线：目标角色明确，经历递进，摘要与项目一致。
- JD 匹配：must-have 有证据，nice-to-have 不冒充已掌握。
- 项目背景：说明用户、业务问题、约束和项目阶段。
- 个人贡献：区分主导、实现、协作、维护与团队成果。
- 技术难点：包含失败模式、约束或复杂性，而非技术栈罗列。
- 产物：可识别的系统、模块、接口、工具、流程或文档。
- 结果：技术、业务或组织影响有因果链。
- 指标：口径、基线、时间窗、规模与数据来源可解释。
- 面试可讲性：候选人能解释架构、权衡、故障与边界。
- 一致性：日期、职位、术语、语言时态和数字一致。
